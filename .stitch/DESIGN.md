# NXT//LINK Design System

## Core Identity
- Product: NXT//LINK
- Mission: Turn scattered market and industry signals into a clear operating view for logistics, supply chain, and innovation teams.
- Audience: Operations leaders, economic development teams, strategic scouting, vendor evaluation teams.
- Voice: Direct, informed, calm, credible.

## Visual Language
- Vibe: Futuristic, operational, high-trust, data-dense without feeling chaotic.
- Tone: Bloomberg Terminal discipline mixed with a modern command center.
- Mood: Quiet urgency, not hype.

## Color Philosophy
- Background: Near-black with subtle blue-violet light, never flat pure black everywhere.
- Surface: Layered dark panels with visible borders so density still feels organized.
- Accent: Electric indigo for primary calls to action and active states.
- Support accents: Green for live/healthy, amber for watch items, cyan for data emphasis, red for alerts.
- Text: Bright neutral headline text, softer gray body copy, dim monospace labels for metadata.

## Typography
- Headlines: Space Grotesk, bold, compact, high contrast.
- Data and labels: IBM Plex Mono.
- Copy: Clean sans-serif body that stays readable in dense dashboards.
- Style: Short headlines, plain language, minimal marketing filler.

## Layout Rules
- Lead with a high-impact hero section that explains the product in one sentence.
- Follow with a concise operating summary before deeper grids or dense data.
- Use responsive card grids with subtle elevation and visible borders.
- Keep sections in this order when possible: orientation -> evidence -> action.
- Prefer generous spacing around dense data blocks so the site breathes.

## Component Patterns
- Navigation: Sticky top bar with compact labels and glass surface treatment.
- Hero: Strong headline, one primary action, one secondary action, one quick "what this does" panel.
- Cards: Softly rounded, bordered, low-glare surfaces.
- Status indicators: Monospace labels, tiny uppercase metadata, restrained motion.
- Data strips and tickers: Use for live context, not as the first explanation.

## Interaction Rules
- Motion should be purposeful and sparse.
- Hover states should feel precise, not playful.
- New visitors should always have an obvious first click.
- Avoid decorative copy that does not help orientation or decision-making.
